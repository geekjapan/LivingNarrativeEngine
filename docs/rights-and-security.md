# 権利・セキュリティ上の注意

LivingNarrativeEngineはローカルでの個人利用を基本に設計されています。この文書は法的助言ではありません。公開・販売・共同制作に使う場合は、利用する素材、生成provider、公開地域に適用される最新の契約と法令を利用者自身で確認してください。

## 画像生成と権利

画像生成では、入力した文章や参照画像、出力画像について、著作権、商標権、肖像権、パブリシティ権、プライバシーを侵害しないことを確認してください。実在人物、存命作家の作風、既存作品のキャラクターやロゴを無断で模倣する指示は避け、参照画像は自作物または必要な許諾を得たものだけを使います。

生成物を利用できる範囲、入力データの保持・学習利用、補償の有無、年齢・地域・用途の制限はproviderと契約プランごとに異なり、変更され得ます。画像providerを接続する前と成果物を公開する前に、その時点の利用規約、コンテンツポリシー、商用利用条件を確認し、生成履歴と採否の記録を残してください。生成されたこと自体は、第三者の権利を侵害しないことや、利用者が著作権を取得できることを保証しません。

## ローカル運用の前提

標準構成は単一利用者のローカル環境を前提とし、オンラインTRPGや信頼できない複数利用者向けの認可・分離機能を提供しません。Webサーバーはloopbackへbindし、`workspace/`と`projects/`を共有フォルダーや公開Web領域へ置かないでください。外部公開が必要な場合は、このアプリケーションだけに依存せず、認証、TLS、アクセス制御、監査、バックアップ暗号化を備えた別の境界を設けます。

プロジェクトとバックアップには、物語本文だけでなく、未公開設定、prompt、実行履歴が平文で含まれます。端末のアカウント権限とディスク暗号化を利用し、不要になったexport・backup・run artifactも同じ機密度で削除してください。

## API secretの扱い

API keyやtokenは環境変数または追跡対象外のローカル`.env`から渡し、`project.yaml`、template、prompt、ログ、issue、スクリーンショットへ書き込まないでください。`.env.example`にはダミー値だけを置き、リポジトリへsecretをcommitしません。漏洩が疑われる場合は、ファイルから消すだけでなくprovider側で直ちに失効・再発行し、Git履歴、backup、artifactへの残存も調べます。

外部providerへ送るpromptには、物語上の情報が含まれます。providerのデータ保持、学習利用、保存地域を確認し、機密性が必要なプロジェクトでは外部送信しないmockまたは承認済みのローカルproviderを選びます。エラー報告を共有するときも、key、endpointのcredential、prompt本文を伏せてください。

## GM秘密情報の扱い

`gm_vault.yaml`、sceneの`hidden_facts`、characterの`secrets`と`private_mind`はGM専用情報です。leak checkerは完全一致などの機械的な漏洩を検出する補助であり、言い換え、推測可能な断片、画面共有、生成providerへの送信を完全には防げません。公開前にはreader向け出力を人が確認し、GM画面、端末出力、run artifactをプレイヤーと共有しないでください。

秘密を公開設定へ移す場合は、直接YAMLを書き換えず、公開の根拠とturnを記録する正規の`StateDiff`経路を使います。templateに実在のcredentialや個人情報を秘密として入れないでください。
